## Phase 1: 버그 수정 (즉시) — 현재 세션

### P1-1. generator.ts separator 로직 수정 (Critical)
- **파일**: `src/hwpx/generator.ts:74`
- **문제**: `!row.includes("---") === false` — 논리 오류로 separator 행이 테이블 데이터에 포함됨
- **수정**: 조건을 `if (/^[\s|:\-]+$/.test(row))` 단일 조건으로 간소화
- **검증**: `markdownToHwpx()` 테이블 변환 테스트

### P1-2. walkSection 재귀 깊이 제한 (Critical)
- **파일**: `src/hwpx/parser.ts:484-583` (walkSection), `586-625` (walkParagraphChildren)
- **문제**: XML DOM 재귀에 깊이 제한 없음 → 악성 파일로 스택 오버플로
- **수정**: `depth` 파라미터 추가, MAX_XML_DEPTH=200 초과 시 조기 리턴
- **검증**: 기존 테스트 통과 + 보안 테스트에 깊이 제한 케이스 추가

### P1-3. mergeKoreanLines 테이블 행 보호 (Critical)
- **파일**: `src/pdf/parser.ts:1124-1145`
- **문제**: `|` 시작 테이블 행이 이전 문단에 병합될 수 있음
- **수정**: `startsWithMarker`에 `|` (파이프) 패턴 추가, 또는 curr가 `|`로 시작하면 병합 차단
- **검증**: 테이블 행 직전에 한글로 끝나는 문단이 있는 케이스

### P1-4. CLI --format 값 검증 (Medium)
- **파일**: `src/cli.ts:19`
- **문제**: 잘못된 format 값 무시, markdown으로 폴백
- **수정**: `.choices(["markdown", "json"])` 추가
- **검증**: 잘못된 값으로 CLI 실행 시 에러 메시지 확인

### P1-5. PDF 변수 섀도잉 수정 (Medium)
- **파일**: `src/pdf/parser.ts:432`
- **문제**: 내부 `textItems`가 외부 `textItems` 섀도잉
- **수정**: 내부 변수명을 `cellItems`로 변경
- **검증**: 빌드 통과

## Phase 2: 보안 강화 — 별도 세션

### P2-1. ZIP 전체 파일 누적 크기 추적
- **파일**: `src/hwpx/parser.ts:169-177`
- **문제**: 섹션 XML만 누적 체크, 비섹션 파일 미추적
- **수정**: metadata/header/manifest 파일 async 호출마다 누적 추적
- **검증**: 보안 테스트에 비섹션 파일 크기 초과 케이스

### P2-2. Webhook URL 스킴/호스트 검증 (SSRF 방지)
- **파일**: `src/watch.ts:100`
- **문제**: URL 검증 없이 fetch() 직접 호출
- **수정**: http:/https: 스킴만 허용, localhost/private IP 차단
- **검증**: 차단된 URL로 webhook 설정 시 에러 확인

### P2-3. 하이퍼링크 URL 살균 (XSS)
- **파일**: `src/table/builder.ts:139-140`
- **문제**: `javascript:` URL이 마크다운에 그대로 출력
- **수정**: http/https/mailto 스킴만 허용, 그 외 href 제거
- **검증**: `javascript:alert(1)` href가 제거되는지 확인

### P2-4. isPathTraversal 널바이트 처리
- **파일**: `src/utils.ts:43-46`
- **문제**: `\x00` 바이트 미검출
- **수정**: `normalized.includes("\x00")` 추가
- **검증**: 널바이트 포함 경로 테스트

### P2-5. Levenshtein 길이 가드 추가
- **파일**: `src/diff/text-diff.ts:27`
- **문제**: 긴 문자열 비교 시 O(m*n) CPU 폭발
- **수정**: 문자열 길이 합이 10000 초과 시 길이 차이 기반 빠른 추정
- **검증**: 10000자 이상 문자열 비교 성능 테스트

### P2-6. 깨진 ZIP 스캔 개선 (break → scan forward)
- **파일**: `src/hwpx/parser.ts:327`
- **문제**: 첫 비PK 바이트에서 break → 중간 손상 이후 파일 유실
- **수정**: break 대신 pos++ 후 다음 PK 시그니처 탐색
- **검증**: 중간에 손상된 ZIP 복구 테스트

### P2-7. PDF getDocument 타임아웃
- **파일**: `src/pdf/parser.ts:46`
- **문제**: 악성/대용량 PDF에서 무한 대기
- **수정**: AbortController + 30초 타임아웃
- **검증**: 타임아웃 동작 확인

### P2-8. DOMParser 에러 핸들러 설정
- **파일**: `src/hwpx/parser.ts` 전반
- **문제**: xmldom이 malformed XML을 조용히 복구
- **수정**: errorHandler 설정으로 파싱 경고 수집
- **검증**: malformed XML 입력 시 경고 발생 확인

## Phase 3: 핵심 기능 (v1.7) — 별도 세션

### P3-1. 이미지 추출 (HWP/HWPX)
- HWPX: ZIP 내 이미지 바이너리 추출 → base64 또는 파일 참조
- HWP5: BinData 스트림에서 이미지 바이너리 추출
- IRBlock에 image 타입 실제 데이터 포함
- `![image](data:image/png;base64,...)` 마크다운 출력

### P3-2. PDF 머리글/바닥글 필터링
- 페이지간 반복 텍스트 감지 (Y좌표 상/하단, 텍스트 유사도)
- ParseOptions에 `removeHeaderFooter: boolean` 추가
- 제거된 요소 ParseWarning 기록

### P3-3. 페이지별 부분 파싱 (graceful degradation)
- 한 페이지 파싱 실패 시 해당 페이지만 경고, 나머지 계속
- ParseWarning에 PARTIAL_PARSE 코드 추가
- 현재 all-or-nothing → 부분 성공 지원

### P3-4. 파일 경로 입력 지원
- `parse(filePath: string, options?)` 오버로드
- 스트리밍까지는 아니더라도, 대용량 파일 메모리 효율 개선
- CLI와 라이브러리 API 일관성

### P3-5. CLI 진행률 표시
- `[3/15 pages]` 형태의 진행률
- ParseOptions에 onProgress 콜백 추가
- 멀티 파일 배치 시 `[2/10 files]` 표시
